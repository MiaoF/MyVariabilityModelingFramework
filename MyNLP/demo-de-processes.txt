1.1	UC_GOODS_IDENTIFICATION_100
Vorbedingung:
Eine Avise wurde angelegt.
Mindestens eine Avisenposition wurde erzeugt und hat den Status ready oder processing.
Ablauf:
1.	Benutzer öffnet den Wareneingangsdialog.
2.	Benutzer drückt auf Aktualisieren, um alle avisierte Positionen anzuzeigen.
3.	Der Benutzer wählt eine avisierte Position per Mausklick aus.
4.	Das System ermittelt die noch offene Menge für diese Avisenposition und zeigt diese im Feld Restmenge an.
5.	Der Benutzer wählt einen Transport-Typ aus
6.	Der Benutzer legt für die Avisenposition eine bestimmte Menge zum Einlagern fest.
7.	Der Benutzer wählt einen Einlagerplatz aus.
8.	Der Benutzer drückt den Button „TU bilden“.
9.	Das System(Client) liefert die Informationen TUTyp, Eingelagerte Menge und Wareneingangsplatz an den Server
10.	Das System(Server) erzeugt einen einzigartigen TuName.
11.	Das System(Server) erzeugt eine Tu mit dem erstellten TuNamen.
12.	Das System(Server) erhöht die Anzahl der gelieferten Menge in der Avisenposition
13.	Das System(Server) meldet das erfolgreiche Anlegen einer TU mittels Übergabe des  TuName der erzeugten Tu an den Client
14.	Das System(Server) ermittelt den Drucker und druckt anhand des übergebenen TuNamen das Etikett aus (siehe 1.3 „Etikett drucken“)
15.	Das System(Client) zeigt ein Fenster mit dem erstellten Tunamen an.
16.	Der Benutzer klebt das ausgedruckte Etikett auf die Transporteinheit.
17.	Der Benutzer bestätigt die Meldung durch Drücken des Ok-Buttons
18.	Das System(Client) gibt die Information an den Server weiter.
19.	Das System(Server) fährt die TU ins Lager.

Nachbedingung:
       Der Wareneingangdialog ist zu sehen

       Der Wareneingangdialog ist zu sehen

Vorbedingung:
Der Benutzer hat eine AvisenpositionsId, welche im System nicht vorhanden ist
Ablauf:
1.	Benutzer öffnet den Wareneingangsdialog.
2.	Benutzer drückt auf Aktualisieren, um alle avisierte Positionen anzuzeigen.
3.	Der Benutzer nutzt einer der Filter für Id der Avisenposition, gibt einen Wert ein welcher nicht existiert
4.	Das System zeigt die leere Ansicht an
5.	Das System lässt die Buttons grau bzw. funktionslos außer Aktualisieren, Schließen und Neu.
Nachbedingung:
       Der Wareneingangsdialog ist zu sehen

14.	Das System(Client) zeigt eine Meldung mit dem erstellten TuNamen an.
15.	Der Benutzer klebt das ausgedruckte Etikett auf die Transporteinheit.
16.	Der Benutzer bestätigt die Meldung durch Drücken des Ok-Buttons

18.	Das System(Server) fährt die TU ins Lager.
19.	Das System(Client) Zeigt eine negative Restmenge im Feld Restmenge an.
Nachbedingung:
       Der Wareneingangsdialog ist zu sehen
